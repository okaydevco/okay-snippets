# OKAY DEV® Snippets — HTML

This folder holds small, self‑contained HTML snippets.

## What belongs here
- Semantic structures (headers, nav, main, footers)
- Forms and inputs (validation, accessibility patterns)
- Meta, SEO, favicons, responsive markup
- Reusable HTML patterns that stand alone without a build step

## Snippet guidelines
- Keep it small, focused, and easy to copy/paste
- Add a brief description at the top of the file
- No secrets, tokens, or proprietary code
- Licensed under MIT; optional attribution header allowed
- See the root docs: [README](../README.md), [CONTRIBUTING](../CONTRIBUTING.md), [LICENSE](../LICENSE)

## Recommended snippet format
1. Title (short and descriptive)
2. Description (1–3 lines: what, why, when)
3. Usage notes (optional)
4. The snippet (minimal and production‑friendly)

### Optional attribution header (choose the right comment syntax)
```text
<!-- Copyright (c) 2025 Your Name -->
<!-- Licensed under the MIT License. -->
```

### Example skeleton
```html
<!-- Title: Accessible Skip Link -->
<!-- Description: Keyboard users can jump straight to main content. -->
<!-- Licensed: MIT. Attribution optional per CONTRIBUTING.md -->

<a class="skip-link" href="#main">Skip to content</a>
```
